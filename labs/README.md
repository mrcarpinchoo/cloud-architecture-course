# Labs

This directory contains all hands-on labs for the Cloud Architecture course.

## Structure

Each lab is organized in its own folder with the following naming convention:

```
lab-XX-topic-name/
```

Where XX is the lab number (01, 02, etc.) and topic-name is a brief description of the lab content.

## Lab Schedule

| # | Lab Title | Description | Duration |
|---|-----------|-------------|----------|
| 01 | Introduction to AWS IAM | Creating users, groups, and policies | 40 min |
| 02 | Build your VPC and Launch a Web Server | Setting up networking and deploying a web server | 30 min |
| 03 | Introduction to Amazon EC2 | Launching and managing EC2 instances | 35 min |
| 04 | Working with EBS | Managing block storage for EC2 instances | 30 min |
| 05 | Build a Database Server | Setting up and configuring database services | 30 min |
| 06 | Scale & Load Balance your Architecture | Implementing auto scaling and load balancing | 30 min |

## Lab Requirements

To complete these labs, you will need:

1. An AWS account (Free Tier eligible)
2. A modern web browser
3. Basic understanding of cloud concepts
4. Completion of the corresponding lecture material

## Submission Guidelines

For each lab, you should submit:

1. A completed lab report using the provided template
2. Screenshots of key steps and results
3. Answers to the reflection questions
4. Any code or configuration files created during the lab

## Important Notes

- Always terminate or delete resources after completing labs to avoid unexpected charges
- Follow AWS best practices for security (use IAM roles, don't share credentials)
- If you encounter issues, check the troubleshooting section in each lab guide
- Labs are designed to be completed within the AWS Free Tier when possible
